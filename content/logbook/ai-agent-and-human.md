title: 人工智能和工人智能
date: 2026-05-15
---
哎，说来惭愧，昨天晚上后端同学联系我，指出了一个 bug，需要改改前端代码。代码应该是两周之前写的了，当时为了简化流程，很多东西都是 ai agent 干的，工人智能只是草草地 review 了一下，简单测了一下，没啥问题就上线了。

不过看起来，确乎是堆了一吨**技术债务**了

表现在打开代码库，虽然只隔两周，但是却有点拔剑四顾心茫然，有点不知所措了。一句话[^ 这句真像 ai 生成的]，**我看不懂两周之前写的代码了**，或者更准确的说，我可能并不能精准的定位业务逻辑的位置。归咎于人类智能的记忆减弱或自然消退自然是合理的，不过确乎有点耍无赖的气质在里面，可能 agent 也 contribute to this phenomenon a lot.[^ 这中英文混杂好 llm...]

不过 VS Code 的全文搜索还是好的，搜了几个关键词就找到对应的代码片段了，修修补补一下，好歹是没啥问题了。

前段时间网上流传一张图，大致讨论 Vibe Coding 下几种程序员流派。仔细想想，曾经还是手搓代码[^ 或者说做 CV 工程师，偷取别人的科技]；再后面用上了 GitHub Copilot，自动补全相当爽；再后面靠 Cursor 生成大段业务逻辑，自己只控制技术栈和整体结构；最后成为 cc 大人的狗，彻底扔掉 IDE，沦为麻木的 vibe coder。

![Vibe Coding 下几种程序员流派](/static/tech/ai-human.webp)

哎，不过感觉自己还是一个**半维新派**到**维新派**之间的过渡吧。完全不用 AI 写代码似乎效率和智能都达不到，完全丢弃 IDE 目前来看也不现实。至于纯粹的 Vibe Coder，完全不学习理论知识，直接靠伟大的自然语言写出看起来能跑的屎山，那才是应该完全摒弃的作风。

哎，固然 AI Agent 很强，发展的也很猛，但是我不认为工人智能可以被抛弃，工人智能也不能完全抛弃 IDE，完全丢弃对代码的掌控权，否则，被丢弃的只能是工人智能自己，或者说：工人智能被人工智能的屎山淹没。

最后，总结诸多群友的话：

> 哎，我发现用集成大脑写东西不耗 token，也少 bug
