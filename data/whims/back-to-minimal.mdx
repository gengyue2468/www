# 何为复杂?

复杂是个复杂的概念，复杂到何为复杂这个简单的问题就无比复杂。复杂是无数页面的交织，是绚丽的霓虹灯般交织的色彩，是大块大块文字轰炸的炸弹。复杂指的是不简单，而不简单就是复杂。

# 何为简单？

简单是个简单的概念，简单到人们只需要看一眼就知道简单。小时候我们在做数学题的时候，看一眼就觉得很简单，做着做着就算逐渐复杂，在思路豁然开朗，最终解出的那一刻也会脱口而出“原来这麽简单啊”。简单，简约的设计风格，同样的，无疑是未来的潮流。在越来越快节奏的生活中，人们渴求快速获得信息，此时，简约的设计可以让人们集中注意在他们需要的信息上，而不是受到外界无关信息的过度干扰。同时，简单的设计还可以给人们身心以愉悦，如果在配上少量的、优雅的、自然的动画过渡，这样带来的美妙体验是无与伦比的。

[Apple](https://apple.com)就是这方面的典范。

# 历程

我在[关于](/about)页面提到，我从2020年开始便逐渐接触一些网站建设相关知识，那是我接触到的不过是 [Hexo](https://hexo.io) 和 [GitHub Pages](https://pages.github.com)一类的建站技术。尽管如此，Hexo NEXT 主题的简约化设计依旧在我心里留下了不可磨灭的印象。在这之后一段时间，通过 [Vercel](https://vercel.com/) 认识了 [Next.js](https://nextjs.org)。随后，在 [Lee Robinson](https://leerob.com)、[Witt](https://unix.bio) 等人为代表的 inspire 下，逐渐尝试写一些简单的 Next.js 静态页面。它们的简约、优雅的设计风格深深吸引了我，启发我进行后续的设计，最后引领我走向计算机科学与技术这条道路。

# 期待

我之所以写这篇文章，是因为我有一个期待。我期待在我四年的本科时间里，把这篇文章“回归简单 2025”视作一个起点和一个目标，希冀我和我的网站设计一样，逐渐的回归简单，无论是走向 Homer Simpson 还是 Peter Griffin 那样的极端。不论怎样，那样的人只会在动画片中出现，而我们要做的，只是做一个纯真的人，要多简单就有多简单。

希望这套设计可以从 2025 年起一直存留，不论是大修还是小补，我希望这套设计的本色永不褪色——简单
